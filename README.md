# hthwiki-journal

## 项目简介

hthwiki-journal 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定的域名或网站，而是作为一组独立 HTML 文件的存储与分发集合。这些页面可以用于展示、存档、记录或其它用途，每个页面保持独立完整性，无需额外服务器支持即可在浏览器中直接运行。

## 目录结构

```
hthwiki-journal/
├── README.md          # 本说明文件
├── pages/             # 存放所有 HTML 页面文件的目录
│   ├── example1.html  # 示例页面 1
│   ├── example2.html  # 示例页面 2
│   └── ...
└── assets/            # 可选：存放页面引用的静态资源（如 CSS、JS、图片等）
    └── ...
```

- `pages/`：核心目录，所有 HTML 页面存放于此。每个页面均为独立文件，文件名通常反映页面内容或主题。
- `assets/`：可选目录，用于存放页面可能引用的外部资源，便于管理和复用。

## 页面归档说明

- 本仓库中的每个 HTML 页面均为自包含的独立文件，可直接在浏览器中打开使用。
- 页面内容可能涉及不同主题、记录或信息，具体用途由页面自身定义。
- 页面之间无依赖关系，更新或删除某个页面不会影响其他页面。
- 归档的页面可能随时间更新或补充，建议以仓库中最新版本为准。

## 维护说明

- **添加新页面**：在 `pages/` 目录下新建一个 `.html` 文件，确保其内容完整、可正常显示。如有必要，可将引用的资源文件放入 `assets/` 目录。
- **更新现有页面**：直接修改对应 HTML 文件，并提交变更。建议在提交信息中简要说明修改内容。
- **删除页面**：若某页面不再需要，可直接删除对应文件。删除前请确认不影响其他页面（本仓库页面相互独立，通常无影响）。
- **贡献**：欢迎通过 Pull Request 提交改进或新增页面。提交前请确保文件格式正确，内容符合仓库定位。
